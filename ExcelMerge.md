# ExcelMerge 전용 규칙

이 파일은 `ExcelMerge` 폴더에서 작업할 때 `CLAUDE.md`와 함께 적용되는 규칙입니다.
아키텍처·빌드·함정 기록은 메모리 `excelmerge-project.md` 참고. 이 문서는 **현재 기능 사양**만 다룹니다.

## 개요
Excel(xlsx/xls/xlsm/xlsb)·CSV/TSV·JSON·uasset을 A/B로 비교하고 셀 단위로 병합하는 PyQt5 데스크톱 툴.
(CSV/TSV·JSON·uasset은 **비교 전용** — 저장/병합은 Excel만 지원.)
진입점 `excel_diff_merge.py` + `excelmerge/` 패키지(loaders·diff_engine·diff_model·widgets·panels·
main_window·folder_compare·folder_view·workers·xlsx_writer·theme·uasset_parser).

## 탭/창
- 상단 탭 스트립 컨테이너, 탭마다 비교 뷰(X 버튼 닫기). 우상단 `＋ 새 비교` 하나(파일/폴더 미구분).
- 시작 시 빈 탭 1개 — 파일 드롭/선택 시 셀 비교, 폴더면 그 탭이 폴더 비교로 전환.
- 같은 (A,B) 경로는 중복 없이 기존 탭으로 전환. 타이틀바 `ExcelMerge v<버전>` 고정(파일/폴더명은 탭 라벨).
- 탭 라벨 아이콘 — 형식별 PNG(엑셀·uasset=Excel.png, JSON=JSON.png), 폴더 탭은 📁.
- 단축키 — Ctrl+Tab/Ctrl+Shift+Tab(다음/이전, 순환), Ctrl+PageDown/PageUp(동일), Ctrl+W(닫기).
- 모든 탭이 하단 상태바 공유(활성 탭 갱신), 단축키는 활성 탭에만.

## 파일/폴더 로딩
- 드래그앤드롭 / 경로칸 직접 입력(Enter) / 찾아보기(네이티브 대화상자) — 모두 파일·폴더 자동 판별.
- 커맨드라인 `-s <A> -d <B>`(P4V 등) — 폴더면 폴더 비교, 파일이면 셀 비교로 진입.
- 빠른 로딩 — 값은 python-calamine(Rust, openpyxl 대비 3~5×, A/B 병렬), 시트명은 workbook.xml만
  읽어 취득. calamine 실패 시 openpyxl 폴백. (캐시값 기반 — 미캐시 수식 셀은 빈 값일 수 있음)

## 폴더 비교
- Beyond Compare 스타일 — 좌/우 트리에 A/B 파일 목록. 하위 폴더 재귀, 상대경로 매칭(대소문자 무시).
- 동일/변경 판정 — 바이트 스트리밍 비교(첫 불일치 즉시 중단) 먼저, 다르면 지원 형식만 **내용 비교**로
  재확인(재저장 등 바이트만 다른 오탐 제거). 파일쌍 판정은 스레드 풀 병렬.
- 스캔은 **백그라운드 실행**(UI 안 얼림), 상태바에 진행 표시("폴더 비교 중… 342/1,200"). 결과는
  (경로·mtime·크기) 캐시로 재사용 → 새로고침·병합 직후 재스캔이 빠름(안 바뀐 파일은 재비교 생략).
- 상태색 — 🟢신규(한쪽만) / 🟡변경, 반대쪽 빈 칸은 대각선 해치. 범례 신규/변경/준비/병합.
- "변경점만 보기" 기본 ON(동일 숨김), 이전/다음 변경 이동·파일명 찾기, 좌우 트리 스크롤·선택 연동(다중 미러).
- 더블클릭/Enter → 파일 비교 탭(다중 선택 Enter는 각각 탭). 한쪽만 있으면 미리보기.
- 파일 병합 — 우클릭 'A→B / B→A 병합 준비'(다중), 패널별 '병합 실행' 버튼으로 복사(확인창).
  준비=주황, 완료=연파랑('변경점만 보기'에서도 유지).

## 시트 선택
- 창 하단 공용 시트 탭(A∪B 합집합·A 순서 우선). 첫 시트 디폴트, 탭 클릭 시 재비교.
- 한쪽에만 있는 시트는 반대쪽 빈 상태. 시트 1개거나 CSV/TSV·JSON/uasset이면 탭 숨김. 저장은 선택 시트에 기록.
- **변경 시트 탭 노랑 표시** — 두 파일의 해당 시트가 서로 동일하지 않으면(값 매트릭스 비교, 키 무관 —
  폴더 diff 방식; 한쪽에만 있는 시트 포함) 탭 배경을 노랑으로. 판정은 백그라운드(SheetDiffWorker).

## 비교(Diff)
- A/B 모두 지정 시 자동 비교. 키 열 기반(지정 열로 행 매칭, 기본 A열) / 키 해제 시 ROW 순서 1:1.
- 키 헤더 앵커 = (키 행, 키 열). 키 행 = 헤더 행 — 그 위 행은 위치 기준 1:1(프리앰블), 아래부터 키 열로 매칭.
  기본 A1. 사용자가 키 행/열을 바꾸면 그 위치를 전역 저장(%APPDATA%/ExcelMerge/prefs.json)해 이후 기본값으로 적용,
  시트 범위를 벗어나면 A1로 자동 리셋(자동 리셋은 저장 안 함).
- **틀 고정(엑셀식, 항상 켜짐)** — 앵커까지의 행/열(0..키행 상단, 0..키열 좌측)을 스크롤해도 고정 표시.
  본체가 고정 행/열을 숨겨 스크롤 영역만 렌더(가림 없음), 좌상단 corner 뷰가 고정 눈금('2'/'B')을 정확히 표시.
  구현: `FreezeController`(widgets.py) + `ExcelTableView.updateGeometries()` 여백 예약 + `ScrollPerItem`(정렬용).
- 셀 색 — 🟢신규(값 있는 쪽 패널만 초록, 반대쪽 해치) / 🟡변경 / 🟠준비 / 🔵병합(저장 후에도 유지).
- "변경점만 보기"(Ctrl+D, 기본 ON), 새로고침(F5), 이전/다음 변경(Alt+Up/Down).
- A/B 문자 차이 강조 — modified 셀의 다른 구간을 핑크 배경(#FFC9C9)+빨강 폰트(#C62828).
  그리드·셀값란 공통(`diff_model.diff_char_ranges` 단일 출처).
- 수식 결과 셀 — 수식으로 계산된 값 셀은 파랑 폰트(#1565C0)로 구분(값 자체는 계산값), 변경 구간은
  빨강 우선. 수식 여부는 첫 셀 선택 시 시트 XML `<f>` 스캔으로 지연 로드.

## 병합(Merge)
- 병합 준비 — 셀 우클릭 / 행 헤더 우클릭(행 전체) / 열 헤더 우클릭(열 전체)로 A→B·B→A 지정.
  우클릭으로 취소, Ctrl+Z 단계 되돌리기.
- 값으로 병합 — 수식이 아닌 계산값을 기록(값 붙여넣기).
- 서식 보존 — 값과 함께 셀 서식(표시형식·폰트·채우기·테두리·정렬) 복사. 셀 단위만(열/행 너비·
  조건부서식·병합셀 제외).

## 저장
- 패널별 A/B 개별 저장 버튼. 현재 선택 시트에 기록. 대상 파일이 Excel에 열려 있으면 경고.
  행 삭제 시 행 번호 자동 재정렬.

## 헤더 / 셀 UI
- 키 열 헤더 = Key.png + 노란 배경, 키 행(헤더 행) 헤더도 Key.png + 노란 배경(세로 헤더). 검사 제외 열 =
  Exception.png + 회색 배경(셀 배경도 회색). 아이콘은 흰 배경 투명 처리.
- 열 헤더 우클릭 — 키 열엔 '키 열 해제'만, 비-키 열엔 '키 열로 설정'(키 아이콘). 다중 열도 병합 준비/취소.
  검사 제외/해제 토글(키 열 제외 불가, 제외·비제외 섞이면 둘 다 노출). 제외/해제 시 스크롤 좌상단 초기화.
- 행 헤더 우클릭 — 단일 행에 '키 행으로 설정'(키 아이콘)/현재 키 행엔 '키 행 초기화(1행)'. 병합 준비/취소도.
  (display 행은 재정렬될 수 있어 원본 파일 행 번호로 지정된다.)
- 셀 값 표시란(읽기전용) — 선택 셀의 **값** 표시(수식 셀도 계산값). 기본 4줄, 스플리터로 조절(A/B 동기).
  A/B 다르면 다른 구간 강조.
- 양쪽 스크롤·선택 셀 동기화. 상태바에 비교 결과(행·열·변경 셀 수) 표시.

## 찾기
- 검색란 상시 노출(Ctrl+F 포커스). Enter=다음, Shift+Enter=이전, ◀/▶ 버튼.
  대소문자 무시(기본 ON)·전체 단어 일치(기본 OFF) 토글.
- 보이는 행만 검색(A/B 표시 텍스트 중 하나라도 일치). 끝에서 순환, 상태바 "n/전체".
  미리보기(단일 파일) 상태에서도 검색 가능.

## 자동 업데이트 (`updater.py`)
- 실행 시 백그라운드로 최신 버전을 조회 → 현재보다 높으면 알림 → 원클릭으로 새 exe를 내려받아
  (진행률·sha256 검증) 자기 자신을 교체·재시작(frozen에서만). 메뉴 `도움말 > 업데이트 확인`/`버전 정보`.
- **소스(둘 중 하나, 없으면 완전 비활성)**:
  - `GITHUB_REPO="owner/name"`(공개 repo) → `releases/latest` API 사용(권장, 매니페스트 파일 불필요).
    `tag_name`=버전(v 접두 허용), `.exe` asset이 다운로드 대상, asset `digest`로 sha256.
  - `MANIFEST_URL`(범용 latest.json 직링크). Drive 대용량 확인페이지·공유링크→직링크 변환 지원.
  - 둘 다 `%APPDATA%/ExcelMerge/update.json`(`github_repo`/`manifest_url`)로 재빌드 없이 덮어쓰기 가능.
- **릴리스(GitHub)**: 버전 bump→빌드→ `gh release create v<N> dist/ExcelMerge_v<N>.exe -R owner/name`
  (또는 웹 UI로 태그 `v<N>` + exe 첨부). `make_release.py`가 sha256·명령을 출력. 첫 버전만 수동 배포.
- 주의: GitHub는 **public repo**여야 미인증 다운로드 가능(private면 토큰 필요 → 미지원).
